## Задание (все подпункты)
### Задание 2.1. Подготовка полигона
Скопировать `~/my_company` в `~/lab2` и перейти в каталог.

### Задание 2.2. Анализ текущих прав
Выполнить `ls -lR` и зафиксировать текущие права (в `initial_perms.txt`) для:
1) `departments/hr/`  
2) `departments/hr/employees.txt`  
3) `projects/project_alpha/`  
4) `shared/scripts/deploy.sh`  
5) `departments/it/servers/`

### Задание 2.3. Символьный режим chmod
Выполнить (фиксировать команду и `ls -l/ls -ld` после каждого пункта):
1) `employees.txt`: владелец rw, группа r, остальные 0  
2) `projects/archive/`: вход только владельцу  
3) `deploy.sh`: исполняемый для владельца и группы  
4) `servers`: группе можно создавать файлы, но не удалять чужие (sticky bit; можно спросить AI)  
5) `project_alpha`: владелец всё, группа r+x, остальные 0

### Задание 2.4. Числовой режим chmod
Заполнить таблицу «права → число» и применить `chmod` для объектов из задания.

### Задание 2.5. Смена владельца (при наличии sudo)
`useradd`, `groupadd`, `chown/chgrp`, рекурсивная смена владельца/группы, проверка `ls -l`.  
Если `sudo` нет — записать команды и ожидаемый эффект.
